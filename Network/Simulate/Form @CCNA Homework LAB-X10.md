# CCNA Homework LAB-X10: Inter-VLAN Routing (Router-on-a-Stick)

**รูปที่ 1:** แผนผังเครือข่าย (Topology)
![[Pasted image 20260424135425.png]]

> [!abstract] 📌 รายละเอียดโจทย์ (Router-on-a-Stick)
> โดยปกติอุปกรณ์ที่อยู่คนละ VLAN จะไม่สามารถคุยกันได้ (แม้อยู่บน Switch ตัวเดียวกัน) แล็บนี้จึงนำ **Router** เข้ามาต่อกับ Switch บนสาย Trunk และสร้าง **Sub-Interface** (อินเทอร์เฟซย่อยเสมือน) เพื่อทำหน้าที่เป็น Default Gateway ให้กับแต่ละ VLAN ทำให้สามารถคุยข้ามกลุ่มกันได้

---

### 🛠 ขั้นตอนที่ 1: เปิดการใช้งานพอร์ตบน Router
**รูปที่ 2:** คำสั่งเปิดพอร์ต Physical
![[Pasted image 20260424135743.png]]

> [!note] ⚙️ การเปิดพอร์ตหลัก (Physical Interface)
> เราต้องเข้าไปที่พอร์ตหลักของ Router (ในที่นี้คือ `g0/0`) แล้วใช้คำสั่ง `no shutdown` เพื่อเปิดให้พอร์ตทำงานก่อนที่จะไปสร้างพอร์ตย่อยด้านใน

---

### 🛠 ขั้นตอนที่ 2: การสร้าง Sub-Interface และกำหนด IP (Gateway)
**รูปที่ 3:** การสร้าง Sub-Interface สำหรับแต่ละ VLAN
![[Pasted image 20260424135805.png]]

> [!info] ⚙️ คำสั่งตั้งค่า Sub-Interface (Dot1Q)
> การสร้างพอร์ตย่อยจะต้องทำ 3 ขั้นตอนหลักต่อ 1 VLAN ดังนี้:
> 1. `interface g0/0.[ID]` : สร้างและเข้าไปยังพอร์ตย่อย (นิยมตั้งตัวเลขให้ตรงกับ VLAN ID)
> 2. `encapsulation dot1Q [VLAN-ID]` : กำหนดชนิดของการห่อหุ้มข้อมูลแบบ 802.1Q และระบุว่าพอร์ตย่อยนี้จะรับผิดชอบ VLAN เบอร์อะไร
> 3. `ip address [IP] [Subnet]` : กำหนดหมายเลข IP เพื่อใช้เป็น **Default Gateway** ให้กับเครื่องใน VLAN นั้นๆ

### 🔍 ขั้นตอนที่ 3: ตรวจสอบสถานะการเชื่อมต่อ
**รูปที่ 4:** ตรวจสอบพอร์ตและตารางเส้นทาง (Routing Table)
![[Pasted image 20260424135815.png]]

> [!success] 📊 การตรวจสอบสถานะบน Router
> - **`show ip interface brief`:** ตรวจสอบพอร์ต จะเห็นว่าพอร์ตย่อย `g0/0.10`, `g0/0.20` และ `g0/0.30` ได้รับหมายเลข IP ตามที่ตั้งไว้และสถานะเป็น **up**
> - **`show ip route`:** ตรวจสอบตารางเส้นทาง จะเห็นว่า Router รู้จักเส้นทางไปยังเครือข่ายย่อยทั้ง 3 วงแล้ว (ตัวย่อ **C** หมายถึง Directly Connected)

### ✅ ขั้นตอนที่ 4: การทดสอบการเชื่อมต่อข้าม VLAN
**รูปที่ 5:** ทดสอบ Ping ข้ามวงเครือข่าย
![[Pasted image 20260424135824.png]]

> [!example] 🚀 ผลการทดสอบ (Ping ข้าม VLAN)
> ทำการทดสอบโดยให้ PC1 (ซึ่งอยู่ใน VLAN 10) ทดสอบ Ping ไปหาอุปกรณ์หมายเลข IP `192.168.20.1` (ซึ่งอยู่ใน VLAN 20)
> - **ผลลัพธ์:** ปรากฏว่า Ping สำเร็จ มี Reply กลับมา (อาจมี Timeout ในแพ็กเก็ตแรกเนื่องจากกระบวนการค้นหา ARP) 
> - **สรุป:** การทำ Router-on-a-Stick สำเร็จลุล่วง อุปกรณ์ต่าง VLAN สามารถคุยกันได้ผ่าน Router แล้ว
