# CCNA Homework LAB-X06: VLAN & Trunking Configuration

**รูปที่ 1:** แผนผังเครือข่าย (Topology)
![[Pasted image 20260424114456.png]]

> [!abstract] 📌 รายละเอียดโจทย์ (Topology)
> แผนผังแสดงการแบ่งกลุ่มเครือข่ายออกเป็น 3 VLAN ได้แก่:
> - **VLAN 10 (Users):** กลุ่มสีฟ้า
> - **VLAN 20 (Sales):** กลุ่มสีน้ำตาล
> - **VLAN 30 (IT):** กลุ่มสีเขียว
> โดยใช้หมายเลข IP Network: `172.16.0.0/16` และมี Switch ทั้งหมด 5 ตัว (SW1-SW5) เชื่อมต่อกัน

![[Pasted image 20260424114633.png]]

> [!note] 🔌 การตั้งค่า Trunking
> ใช้คำสั่ง `switchport mode trunk` บนอินเทอร์เฟซที่เป็นเส้นเชื่อมต่อระหว่าง Switch เข้าด้วยกัน การทำ Trunking จะช่วยให้ข้อมูลของทุกๆ VLAN สามารถวิ่งผ่านสายสัญญาณเส้นเดียวกันนี้ได้ (ถ้าไม่ทำ ข้อมูลจะข้าม Switch ไม่ได้)

> [!success] 🔍 การตรวจสอบสถานะ Trunk

### 🛠 ขั้นตอนที่ 3: การกำหนดพอร์ตสำหรับอุปกรณ์ปลายทาง (Access Port)
**รูปที่ 12 - 13:** การกำหนดพอร์ตให้กับ PC และ Server ตาม VLAN ที่ต้องการ
![[Pasted image 20260424114719.png]]
![[Pasted image 20260424114726.png]]

> [!note] 💻 การตั้งค่า Access Port
> ใช้คำสั่ง `switchport mode access` เพื่อกำหนดให้พอร์ตต่อกับอุปกรณ์ปลายทาง (PC, Server) และใช้คำสั่ง `switchport access vlan [ID]` เพื่อระบุว่าพอร์ตนั้นๆ สังกัดอยู่ VLAN ใด (เช่น พอร์ตที่ต่อกับ PC1 ให้ไปอยู่ในกลุ่ม VLAN 10)

**รูปที่ 14:** ตรวจสอบรายละเอียดของพอร์ต
![[Pasted image 20260424114753.png]]

> [!success] 🔍 การตรวจสอบพอร์ต Access
> ใช้คำสั่ง `show interfaces [Port] switchport` (เช่น `show interfaces fa0/1 switchport`) เพื่อดูรายละเอียดเชิงลึก จะเห็นชัดเจนว่าพอร์ตนี้ถูกตั้งค่าเป็น **static access** และถูกจัดให้อยู่ใน **Access Mode VLAN: 10 (Users)**

> [!example] 🚀 ผลการทดสอบ (Ping Test)

> - **กลุ่ม VLAN 30:** PC3 สามารถ Ping ไปหา Server3 ได้สำเร็จ
> 
> **สรุปผลการทดลอง:** การกำหนดค่า VLAN เพื่อแบ่งกลุ่ม และการทำ Trunking เพื่อเชื่อม Switch เข้าด้วยกัน ทำงานได้อย่างถูกต้อง ทำให้อุปกรณ์ใน VLAN กลุ่มเดียวกันสามารถสื่อสารกันได้ข้าม Switch อย่างสมบูรณ์
