# CCNA Homework LAB-X04

**รูปที่ 1:** โครงร่างเครือข่ายสำหรับการทำแล็บ
![[Pasted image 20260424105654.png]]

> [!abstract] 📌 โครงร่างเครือข่ายสำหรับการทำแล็บ
> แสดงโจทย์เริ่มต้นของแล็บนี้ โดยให้เราใช้เครื่องคอมพิวเตอร์ทำการเชื่อมต่อเข้าไปยังอุปกรณ์เป้าหมายที่อยู่ในกลุ่มเมฆ (CDP Cloud) ซึ่งเรายังไม่ทราบว่าข้างในต่อกันอย่างไร โดยใช้วิธี **Telnet ไปที่หมายเลข IP 172.16.0.252** และใช้ **รหัสผ่าน: cisco**

**รูปที่ 2:** รายละเอียดการเชื่อมต่อเข้าสู่ CDP Cloud
![[Pasted image 20260424105745.png]]

> [!info] 🔍 รายละเอียดการเชื่อมต่อเข้าสู่ CDP Cloud
> ภาพนี้แสดงรายละเอียดที่ชัดเจนขึ้นในโปรแกรมจำลองเครือข่าย (เช่น Cisco Packet Tracer) โดยเครื่อง PC0 เชื่อมต่อผ่านพอร์ต Fa0 ไปยังพอร์ต Fa0/2 ของอุปกรณ์ที่อยู่ในฝั่ง CDP Cloud ซึ่งเป้าหมายหลักของแล็บนี้คือการให้เราเข้าไปที่อุปกรณ์ตัวแรกผ่านทาง Telnet ให้สำเร็จเสียก่อน จากนั้นจึงใช้คำสั่งเกี่ยวกับ CDP เพื่อค้นหาแผนผังเครือข่ายส่วนที่เหลือ

**รูปที่ 3:** การใช้คำสั่งตรวจสอบอุปกรณ์ข้างเคียง (CDP Neighbors)
![[Pasted image 20260424110705.png]]

> [!success] 📡 การใช้คำสั่งตรวจสอบอุปกรณ์ข้างเคียง (CDP Neighbors)
> เมื่อเรา Telnet เข้ามาที่ Switch (SW1) ได้แล้ว เราใช้คำสั่ง `show cdp neighbors` เพื่อสำรวจว่า SW1 เชื่อมต่อกับใครบ้าง ซึ่งจากผลลัพธ์ทำให้เราทราบแผนผังเครือข่ายดังนี้:
> 1. ต่อกับ **Router (R1)** ที่พอร์ต FastEthernet 0/1
> 2. ต่อกับ **Switch (SW2)** สองเส้นทาง (Redundancy) ที่พอร์ต Gigabit 0/1 และ 0/2
> 3. ต่อกับ **Layer 3 Switch (SW-L3)** ที่พอร์ต FastEthernet 0/5
> ข้อมูลเหล่านี้ช่วยให้เราสามารถวาดแผนผังเครือข่าย (Topology) ทั้งหมดออกมาได้แม้จะมองไม่เห็นด้วยตาเปล่าในตอนแรก
